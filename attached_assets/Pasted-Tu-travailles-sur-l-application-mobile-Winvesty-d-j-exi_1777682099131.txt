Étapes obligatoires avant de coder :

1. Vérifier l’état actuel du projet.
2. Créer un commit Git propre de la version actuelle.
3. Créer une branche de sauvegarde.
4. Créer un tag de version stable.
5. Créer un fichier CHANGELOG.md.
6. M’expliquer clairement comment revenir à cette version.

Noms à utiliser :

backup/mobile-app-current-version-before-investor-startup-ux

stable-mobile-app-v1-before-ux-upgrade

Message de commit :
Save current Winvesty mobile app version before investor and startup UX upgrade

git status

git branch backup/mobile-app-current-version-before-investor-startup-ux

APRÈS LA SAUVEGARDE UNIQUEMENT

Tu peux ensuite améliorer l’application mobile Winvesty.

CONTEXTE

L’application mobile Winvesty existe déjà en première version.

Ne repars pas de zéro.
Conserve l’existant.
Améliore progressivement l’expérience investisseur et l’expérience startup pour que l’application ne soit pas une simple version mobile du site, mais un véritable espace privé intelligent.

Winvesty est une plateforme stratégique dédiée à la levée de fonds, au M&A, au matching investisseurs, à la structuration des dossiers, à la visibilité qualifiée et à une Deal Room privée.

Le site public est sur Wix.
Les membres, contacts et collections startups sont actuellement dans Wix.
L’application doit pouvoir communiquer avec Wix via des APIs Velo sécurisées, et préparer progressivement une architecture Supabase.

OBJECTIF STRATÉGIQUE

Transformer l’app en :

1. Investor Cockpit
2. Startup Cockpit
3. Deal Room intelligente
4. Système de matching
5. Système de notifications qualifiées
6. Espace de suivi des dossiers
7. Interface premium, moderne, crédible et réellement app-like

IMPORTANT

L’application ne doit pas être un simple site encapsulé.
Elle doit avoir une vraie expérience mobile :
- splash screen ;
- navigation basse ;
- onboarding ;
- notifications ;
- watchlist ;
- filtres dynamiques ;
- scores ;
- interactions natives ;
- écrans personnalisés ;
- états de chargement premium ;
- feedback utilisateur clair.

PARTIE 1 — IDENTITÉ VISUELLE WINVESTY

Utiliser le logo officiel Winvesty fourni dans les assets du projet.

Ne recrée pas le logo à partir de zéro.
Ne génère pas une approximation du W.
Ne modifie pas la forme du W ni la flèche.
Ne change pas les proportions.

Le logo officiel doit être utilisé pour :
- le splash screen ;
- le header ;
- l’écran de connexion ;
- l’icône interne de l’application ;
- les écrans onboarding ;
- les pages premium de la Deal Room.

Si plusieurs versions du logo sont fournies, utilise :
- la version claire sur fond sombre ;
- la version sombre sur fond clair ;
- l’icône seule W pour les petits espaces ;
- le logo complet Winvesty pour les écrans d’accueil.

PARTIE 2 — INVESTOR COCKPIT

Créer un écran d’accueil personnalisé pour les investisseurs validés.

L’écran doit afficher :

- nombre de nouvelles opportunités correspondant à ses critères ;
- opportunités recommandées ;
- opportunités récemment ajoutées ;
- opportunités confidentielles disponibles sur demande ;
- dossiers sauvegardés ;
- demandes de mise en relation ;
- notifications non lues ;
- accès rapide aux critères d’investissement.

Blocs à créer :

- Recommandé pour vous
- Nouveaux dossiers
- Opportunités confidentielles
- Ma Watchlist
- Mes critères
- Mes demandes de mise en relation

Exemple :

“3 nouvelles opportunités correspondent à vos critères.”

- deck disponible ;
- mandat signé ;
- nouveauté ;
- bouton Voir le dossier ;

Ajouter un badge :

“Correspond à vos critères”
