Winvesty est une plateforme stratégique dédiée à la levée de fonds, au M&A, à la structuration de dossiers, à la visibilité qualifiée, au matching investisseurs et à une Deal Room privée.

OBJECTIF GLOBAL

L’application doit permettre :

- aux startups de s’inscrire ;
- aux startups de déposer leur dossier ;
- à Winvesty de valider une startup ;
- à Winvesty d’indiquer qu’une startup a signé un mandat ;
- à Winvesty de publier cette startup dans la Deal Room ;
- aux investisseurs de renseigner leurs critères d’investissement ;
- aux investisseurs validés d’accéder à une Deal Room mobile ;
- aux investisseurs de recevoir une notification lorsqu’une nouvelle startup publiée dans la Deal Room correspond à leurs critères ;
- aux investisseurs de filtrer manuellement la Deal Room avec des critères précis.

IMPORTANT

Il faut bien distinguer deux types de critères :

1. Critères permanents de l’investisseur
Ces critères sont renseignés dans son profil ou pendant l’onboarding investisseur.
Ils servent au matching automatique et aux notifications.

2. Filtres temporaires de la Deal Room
Ces filtres sont utilisés directement dans la Deal Room mobile.
Ils permettent à l’investisseur de choisir ce qu’il veut afficher à un moment donné.

Les deux systèmes doivent coexister.

PARTIE 3 — NOTIFICATIONS INTELLIGENTES

Créer un système de notifications intelligentes.

Quand une nouvelle startup est publiée dans la Deal Room, l’application doit comparer cette startup aux critères enregistrés de chaque investisseur validé.

Si la startup correspond aux critères d’un investisseur, créer une notification pour cet investisseur.

Exemple de notification :

Titre :
Nouvelle opportunité correspondant à vos critères

Message :
Une nouvelle startup vient d’intégrer la Deal Room Winvesty et correspond à vos critères d’investissement.

Autre exemple plus premium :

Titre :
Nouvelle opportunité qualifiée

Message :
Une société récemment intégrée à la Deal Room correspond à vos critères d’investissement. Consultez son dossier.

Créer une table :

notifications

Champs recommandés :

- id
- user_id
- type
- title
- message
- related_opportunity_id
- is_read
- priority
- created_at

Types de notification :

- new_matching_opportunity
- startup_published
- connection_request_update
- profile_validation
- dealroom_update

Règle de déclenchement :

Lorsqu’une opportunité passe en published_in_dealroom :
1. Récupérer tous les investisseurs validés.
2. Récupérer leurs critères d’investissement.
3. Comparer la startup avec leurs critères.
4. Calculer un score de compatibilité.
5. Si le score dépasse un seuil minimum, créer une notification.
6. Marquer la notification comme non lue.
7. Afficher un badge dans l’application.

Score recommandé :

- secteur correspondant : +30 points
- pays correspondant : +20 points
- ticket compatible : +20 points
- stade compatible : +15 points
- chiffre d’affaires compatible : +10 points
- type d’opération compatible : +15 points
- mots-clés compatibles : +10 points

Si score >= 50 :
créer une notification.

Si score >= 75 :
notification prioritaire.

Ajouter dans l’interface :

- icône notifications ;
- badge rouge ou discret avec nombre de notifications non lues ;
- écran Notifications ;
- bouton “Marquer comme lu” ;
- bouton “Voir l’opportunité” ;
- possibilité de désactiver les notifications dans le profil investisseur.

Préparer le code pour les notifications push Expo plus tard, mais en V1 créer au minimum des notifications in-app stockées en base Supabase.

- pays ;
- secteur ;
- sous-secteur ;
- stade ;
- type d’opération : levée de fonds, M&A, cession, acquisition, rapprochement ;

L’investisseur doit pouvoir :

- ouvrir les filtres ;
- sélectionner plusieurs critères ;
- voir uniquement les startups correspondant aux filtres ;
- réinitialiser les filtres ;
- sauvegarder une recherche ;
- activer une alerte sur une recherche sauvegardée plus tard.

Interface recommandée :

En haut de la Deal Room :

- barre de recherche ;
- bouton “Filtres” ;
- bouton “Correspond à mes critères” ;
- bouton “Réinitialiser” ;
- tri par pertinence, date d’ajout, montant recherché, score de préparation.

Les filtres doivent fonctionner sur les données Supabase.

Par défaut :

- afficher les opportunités recommandées en premier ;
- puis les autres opportunités disponibles ;
- mettre un badge “Match” ou “Correspond à vos critères” sur les opportunités compatibles avec le profil investisseur.
